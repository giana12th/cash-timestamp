# 要件定義書：タイムスタンプ経過時間表示 VSCode拡張機能

## 概要

外部アプリが書き込むタイムスタンプファイルを監視し、現在時刻との差分（経過時間）をVSCodeのステータスバーに常時表示する拡張機能。

---

## 機能要件

### 経過時間の表示

- VSCodeのステータスバーに経過時間を常時表示する
- 表示フォーマット：`HH:MM:SS`（例：`02:30:15`）
- 1秒ごとに表示を更新する

### タイムスタンプファイルの読み込み

- 起動時にファイルを読み込み、経過時間の計算を開始する
- ファイルが更新されたら自動で検知し、表示をリセット・再計算する（`fs.watch` を使用）
- ファイル更新イベントの二重発火対策としてデバウンス処理を行う

### エラー時の表示

- ファイルが存在しない、または読み込めない場合はステータスバーに `--:--:--` を表示する
- エラー解消後（ファイルが生成・修復された場合）は自動で正常表示に戻る

---

## 非機能要件

- Windows専用（他OSの動作は保証しない）
- 自分専用・単一端末での利用（配布・インストーラー不要）
- 表示専用（ファイルへの書き込みは行わない）

---

## 設定

`settings.json` で以下を設定できる。ユーザー設定・ワークスペース設定の両方に対応しており、ワークスペースごとに異なるタイムスタンプファイルを指定可能。

| 設定キー | 型 | 説明 | 例 |
|---|---|---|---|
| `timestampViewer.filePath` | string | タイムスタンプファイルの絶対パス | `"C:\\Users\\user\\timestamp.txt"` |

### ワークスペースごとの使い分け

`.vscode/settings.json` に `timestampViewer.filePath` を記述することで、プロジェクトごとに別のタイムスタンプファイルを参照できる。VSCodeのワークスペース切り替えに連動して自動的に切り替わる。

---

## タイムスタンプファイルの仕様

| 項目 | 内容 |
|---|---|
| 拡張子 | `.txt` |
| 内容 | ISO 8601 ローカル時刻を1行だけ記述 |
| 例 | `2026-05-20T15:30:00` |
| 文字コード | UTF-8 |
| 書き込み者 | 外部アプリ（この拡張機能は読み取り専用） |
| タイムゾーン | Windowsのシステム時刻に準拠（タイムゾーン指定なし） |

---

## 技術スタック

| 項目 | 内容 |
|---|---|
| 言語 | TypeScript |
| フレームワーク | VSCode Extension API |
| ファイル監視 | Node.js `fs.watch` |
| ビルドツール | webpack |
| パッケージ管理 | npm |

---

## ステータスバーの表示仕様

| 状態 | 表示例 |
|---|---|
| 正常（経過時間表示中） | `⏱ 02:30:15` |
| ファイルなし／読み取りエラー | `⏱ --:--:--` |

---

## 対象外（スコープ外）

- 複数ファイルの同時監視
- 経過時間のリセット操作（外部アプリがファイルを更新することでリセット）
- 通知・アラート機能
- 他OS対応
- 配布・マーケットプレイス公開
